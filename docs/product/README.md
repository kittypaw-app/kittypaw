# KittyPaw Product Documentation

This directory defines KittyPaw's user-visible concepts and where each concept
is exposed.

Use these documents when planning or reviewing features:

- `concepts.md`: canonical vocabulary, source of truth, and concept boundaries.
- `surfaces.md`: CLI/setup/web/chat/hosted exposure matrix.
- `architecture-map.md`: product-level runtime and data-flow diagrams.
- `update-checklist.md`: required doc updates when a concept or surface changes.

Rule: if a feature adds a new concept, command, tool, route, or in-chat behavior,
update these docs in the same change.
