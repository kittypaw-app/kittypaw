Quick Start

10분 안에 KittyPaw 실행하기

목표는 많은 기능을 한 번에 켜는 것이 아니라, 로컬 바이너리가 뜨고 LLM provider 와 채널 설정이 저장되며 첫 대화가 가능한지 확인하는 것입니다.

Guide Map
전체 가이드 빠른 시작 스킬 패키지 채널 연결 모델 선택 Reflection 보안 모델 운영 점검

준비물

  • macOS 또는 Linux 터미널. Windows 는 릴리스에서 바이너리를 직접 받는 흐름을 권장합니다.
  • Claude, OpenAI, Ollama, LM Studio 등 사용할 LLM provider 중 하나.
  • Telegram 같은 외부 채널은 나중에 붙여도 됩니다. 첫 실행은 CLI 또는 로컬 Web UI 만으로도 충분합니다.

1. 설치

공식 설치 스크립트로 최신 릴리스를 내려받습니다.

curl -fsSL https://raw.githubusercontent.com/kittypaw-app/kitty/main/install-kittypaw.sh | sh
kittypaw --version

2. setup 실행

kittypaw setup 은 LLM provider, 모델, 기본 채널, 로컬 서버 설정을 한 번에 묻습니다. 이미 설정한 모델을 바꾸고 싶을 때도 같은 명령으로 다시 들어가면 됩니다.

kittypaw setup

3. 첫 대화 확인

setup 이 끝나면 로컬 서버 또는 CLI 채팅으로 짧은 메시지를 보내 응답이 돌아오는지 확인합니다. 여기서 실패하면 스킬을 설치하기 전에 provider key, local endpoint, 모델 이름부터 고치는 것이 빠릅니다.

kittypaw chat
> 오늘 테스트 메시지에 한 문장으로 답해줘

4. 다음에 확인할 것

  • 로컬 LLM 을 쓴다면 tool use 가 가능한 모델인지 확인합니다. 작은 모델은 일반 대화는 되지만 스킬 호출이 불안정할 수 있습니다.
  • 외부 채널을 붙이기 전에는 먼저 CLI/Web UI 에서 스킬 수동 실행이 되는지 확인합니다.
  • cron 발송을 켜기 전에는 kittypaw skill run <name> 으로 출력 형식을 확인합니다.
빠른 성공 기준

kittypaw setup 이 저장되고, 첫 대화가 한 번 성공하고, 로컬 서버가 재시작 후에도 같은 설정을 읽으면 다음 단계로 넘어갈 준비가 된 상태입니다.
