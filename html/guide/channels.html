어떤 채널로 KittyPaw 를 쓸까요?

KittyPaw 는 CLI/Web UI 를 기본으로 두고 Telegram, Slack, Discord, Kakao, WebSocket 어댑터를 실험합니다. 처음에는 하나의 채널만 붙여도 충분합니다.

선택 기준

채널은 기능 차이보다 운영 환경 차이가 큽니다. 개인 자동화와 알림은 Telegram 이 가장 단순하고, 팀 채팅은 Slack/Discord 가 자연스럽습니다. Kakao 는 한국 사용자에게 친숙하지만 외부 relay 와 callback 제약을 더 신중히 봐야 합니다.

CLI / Web UI

처음 검증, 스킬 수동 실행, 실패 메시지 확인에 적합합니다.

Telegram

개인 알림, 아침 브리핑, URL 모니터링처럼 검증된 흐름에 적합합니다.

Slack / Discord

팀 채널에 결과를 흘리거나 운영 알림을 공유할 때 맞습니다.

Kakao / WebSocket

한국 사용자 접점이나 hosted relay 실험에서 사용합니다.

권장 연결 순서

  1. CLI 또는 Web UI 에서 LLM 응답과 스킬 수동 실행을 확인합니다.
  2. Telegram 같은 단일 개인 채널을 붙여 발송 경로를 확인합니다.
  3. 팀 채널 또는 Kakao 는 실패 시 원인 분리가 가능해진 뒤 붙입니다.

채널에서 확인할 것

  • 메시지가 중복 발송되지 않는지.
  • 스킬 실패가 조용히 사라지지 않고 사용자가 볼 수 있는 형태로 남는지.
  • 이미지나 긴 메시지처럼 채널별 제약이 있는 출력은 fallback 이 있는지.
  • 채널 token 또는 webhook secret 이 account 별로 분리되어 저장되는지.
실패 분리 팁

채널 발송이 실패했을 때는 같은 스킬을 CLI 에서 먼저 실행하세요. CLI 출력이 정상이라면 채널 credential, 라우팅, 외부 플랫폼 제한을 보면 됩니다.
