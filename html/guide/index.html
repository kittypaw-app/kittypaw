KittyPaw Guide

설치부터 운영까지 한 번에 보는 가이드

KittyPaw 는 단일 Go 바이너리로 로컬에서 실행되는 AI 에이전트 실험입니다. 이 가이드는 처음 실행하는 사용자, 스킬을 붙이는 사용자, 채널과 권한을 점검하는 사용자가 바로 필요한 문서로 이동하도록 구성했습니다.

$ curl -fsSL https://raw.githubusercontent.com/kittypaw-app/kitty/main/install-kittypaw.sh | sh

🔒 안전하게 운영

sandbox, permission, secrets, 로컬 저장 원칙을 먼저 점검합니다.

가이드 목차

1. 빠른 시작

필수 준비물, 설치 명령, setup wizard, 첫 메시지, 서버 확인.

2. 스킬 패키지

registry 설치, config 입력, manual run, cron trigger, package.toml 읽기.

3. 채널 연결

CLI/Web UI, Telegram, Slack, Discord, Kakao, WebSocket 의 역할과 선택 기준.

4. 모델 선택

기본 모델, named model, /model 전환, cloud/local provider 선택 기준.

5. Reflection

최근 대화 분석, 반복 의도 후보, first-turn suffix, 승인 기반 dispatch.

6. 보안 모델

goja sandbox, SSRF guard, supervised permission, account 별 secrets 저장.

7. 운영 점검

로컬 서버 상태, 스케줄 확인, 알림 실패 원인 분리, 업그레이드 전 체크.

기본 데이터 흐름

사용자는 CLI, 로컬 Web UI, 또는 채널 어댑터로 요청을 보냅니다. KittyPaw 서버는 로컬 SQLite 에 대화와 설정을 저장하고, 필요한 경우 스킬을 goja sandbox 안에서 실행합니다. 스킬 출력은 다시 채팅 응답이나 채널 발송으로 돌아갑니다.
