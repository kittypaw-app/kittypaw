스킬 패키지를 설치하고 실행하는 법

스킬은 KittyPaw 의 확장 단위입니다. registry, GitHub URL, 로컬 path 에서 설치하고 config 를 채운 뒤 수동 실행 또는 cron trigger 로 운영합니다.

Guide Map
전체 가이드빠른 시작스킬 패키지채널 연결모델 선택Reflection보안 모델운영 점검

기본 순서

  1. 패키지를 설치합니다.
  2. package.toml 에 정의된 config key 를 채웁니다.
  3. 수동 실행으로 출력과 실패 메시지를 확인합니다.
  4. 필요하면 cron trigger 와 채널 발송을 켭니다.

1. 설치

레지스트리 패키지는 이름만으로 설치합니다. 실험 중인 패키지는 GitHub URL 또는 로컬 디렉터리로도 설치할 수 있습니다.

kittypaw skill install weather-briefing
kittypaw skill install macro-economy-report
kittypaw skill install rss-digest

2. config 입력

스킬별 필수 config 는 패키지의 package.toml 에 들어 있습니다. API key 가 필요 없는 패키지는 바로 실행할 수 있지만, 위치나 RSS feed 처럼 사용자가 정해야 하는 값은 직접 입력해야 합니다.

kittypaw skill config weather-briefing latitude 37.57
kittypaw skill config weather-briefing longitude 126.98
kittypaw skill config rss-digest feed_url https://example.com/feed.xml

3. 수동 실행으로 검증

cron 으로 보내기 전에 반드시 수동 실행을 먼저 해보는 편이 좋습니다. 이 단계에서 외부 API 실패, config 누락, LLM 요약 실패를 분리할 수 있습니다.

kittypaw skill run weather-briefing
kittypaw skill run rss-digest

4. cron 과 채널

cron trigger 가 있는 패키지는 서버가 실행 중일 때 정해진 시각에 자동 실행됩니다. 발송 대상은 setup 된 채널 어댑터입니다. 채널을 붙이지 않은 상태라면 먼저 CLI/Web UI 에서 결과를 확인하세요.

추천 검증 순서

설치 성공보다 중요한 것은 출력이 실제로 쓸 만한지입니다. skill run 결과가 만족스럽고 실패 메시지가 이해 가능할 때만 cron 을 켜는 흐름을 권장합니다.
