Reflection

반복 요청을 후보로 보여주는 흐름

Reflection 은 최근 대화에서 반복 의도와 토픽 후보를 찾고, 다음 대화 첫 turn 에 사용자에게 제안하는 실험입니다. 자동으로 코드를 바꾸는 기능이 아니라 승인 기반 dispatch 흐름입니다.

Guide Map
전체 가이드빠른 시작스킬 패키지채널 연결모델 선택Reflection보안 모델운영 점검

무엇이 검증됐나요?

  • 최근 대화를 저장하고 반복 의도 후보를 만드는 흐름.
  • chat 첫 turn 에 suffix 로 후보를 surface 하는 흐름.
  • 사용자가 "네"라고 답했을 때 install consent branch 로 dispatch 하는 흐름.

기본 동작

사용자가 비슷한 요청을 반복하면 KittyPaw 는 해당 패턴을 후보로 저장합니다. 예를 들어 환율, 날씨, 사이트 상태처럼 반복되는 요청은 스킬 또는 스케줄 후보가 될 수 있습니다.

사용자: 오늘 환율 알려줘
사용자: 달러 가격 얼마야?
사용자: 환율 매일 확인해줘

KittyPaw: 반복 요청 후보가 있습니다. 환율 조회를 자동화할까요?

승인과 한계

Reflection 은 사용자가 승인하기 전에는 자동으로 설치하거나 코드를 바꾸지 않는 쪽으로 설계되어 있습니다. 자연어 스킬 생성은 syntax 검증이 개선됐지만, 생성된 코드의 런타임 안정성은 패키지마다 다릅니다.

사용자가 점검할 것

  • 제안이 실제 반복 요청인지, 우연히 비슷한 문장 몇 개를 묶은 것인지 확인합니다.
  • 스케줄이 필요한 요청인지, 수동 실행만으로 충분한 요청인지 구분합니다.
  • 외부 API key 나 민감한 데이터가 필요한 자동화는 보안 가이드를 먼저 확인합니다.
좋은 후보
